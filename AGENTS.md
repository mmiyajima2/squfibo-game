# AGENTS.md - Claude Code 作業指針

## 最重要ルール
1. **`./docs/spec_ja.md` の仕様書を必ず最優先で参照する**
2. 仕様に記載がない場合や不明点は、実装前に必ず確認を求める
3. セキュリティ・認証に関わる変更は特に慎重に扱う

## 実装フロー
1. `TODO.md` でタスク内容を確認
2. `docs/spec_ja.md` で該当機能の仕様を確認
3. 仕様に沿って実装
4. 必要に応じてE2Eテストを実装
5. `npm run lint` でLintチェック実行
6. テスト実行(`npm test`)と結果報告
7. 完了したら `TODO.md` を更新

## 技術スタック
- **フロントエンド**: Vite + React + TypeScript
- **Linter**: ESLint
- **テスト**: vitest

## 報告・確認が必要なタイミング
- 仕様書に記載がない機能や判断が必要な時
- 複数の実装方法があり選択が必要な時
- 予期しないエラーや問題に遭遇した時
- 既存コードの大きな変更が必要な時

## コミット前チェックリスト
- [ ] `npm run lint` が通る
- [ ] 該当するテストが通る
- [ ] 仕様書との整合性を確認済み
- [ ] `TODO.md` を更新済み

## エラー対応方針
- エラーログを必ず確認し、根本原因を特定する
- 推測での修正は避け、不明な場合は確認を求める
